why automation?

Automation, in any field, brings in the benefits of increased productivity and reduced costs. It is the creation of technology and its application in order to control and monitor the production and delivery of various goods and services. It performs tasks that were previously performed by humans. Being used in a number of areas such as manufacturing, transport, utilities, defense, facilities, operations and lately, information technology.

Incremental Growth

The first and foremost factor that necessitates automation in agile testing is the short development cycle.

Frequent Changes

The requirements develop with time and often change depending on customer priorities, market trends and end-user needs.

Exhaustive Testing

With automation, testing can be repeated as many times allowing detailed and exhaustive examination of the code.

Automating test support

Automating not only for executing test scripts against the code but can also be used to automate other testing activities like data set up, test result validation and test reporting.

Continuous Testing

Test coverage extends to not only the newly added code but also the code from previous iterations, to ensure previous functionality is not broken due to the newly added one.

Visiblity into code quality

Automation testing allows the tester and developer to take a quick peek into the code quality and they have more time to react in case the code is not up to expectations.
